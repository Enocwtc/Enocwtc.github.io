QuicK Start Guide

Bienvenido a la guía de inicio rápido de myVanilla.js, el micro-framework ultrarrápido diseñado para crear interfaces web dinámicas con la menor cantidad de dependencias posibles y un rendimiento nativo impecable.

Nota de Diseño

myVanilla.js persigue un enfoque minimalista: sin compilation step obligatorio, sin Virtual DOM pesado, y con reactividad basada en Proxies nativos.

Instalación

Para comenzar a utilizar myVanilla en tu proyecto actual, simplemente agrega el framework mediante una clonacion del repositorio.

# Clonar el repositorio desde GitHubaskd
git clone https://github.com/enoc-software/myvanilla-js.git

Primeros Pasos

Insertamos la etiqueta JavaScript

<script type="module" src="assets/js/myVanilla-js/main.js"></script>

Llamada a componentes

Por defecto, los componentes se llaman en el HTML utilizando el prefijo configurado (app- por defecto) seguido del nombre del componente, en minúsculas y separados por un guion (formato kebab-case):

<app-mi-componente></app-mi-componente>

Es fundamental que myVanilla tiene un mequeño archivo de configuración json en la raiz del repo. app-id.json

Guide & Concepts

Esta guía proporciona una visión profunda de los principios arquitectónicos y los bloques de construcción fundamentales que conforman myVanilla.js.

Me inspite en como algunos frameworks crean extructura de datos, como si fuera un standar de la industria facil para que los usuarios familiarizados lo pueda usar.

Los componentes se implementan de forma nativa en el HTML, usando el prefijo configurado (nಠ_ಠn- por defecto) seguido del nombre del componente, en minúsculas y separados por un guion (formato kebab-case).

<nಠ_ಠn-mi-componente></nಠ_ಠn-mi-componente>

Nota :

El prefijo por defecto es nಠ_ಠn-.Puede ser cambiado en el archivo de configuración app-id.json.

No es precompilado, esto quiere decir que el código es interpretado en tiempo de ejecución. Esto permite una mayor flexibilidad y agilidad en el desarrollo, ya que los cambios pueden ser aplicados y probados inmediatamente sin necesidad de un proceso de compilación.

No usa archivos fuera del estandar de la web. y no pretende alterar la forma natural como uno pretenda codear.

Cada componente se renderiza y se cachea. y es reactivo a la hora de acceder a los atributos como Props, Data , Methods

Tambien trabaja con los siclos de vida de los componentes. Created, Mounted , Updated

Reactivity System

A diferencia de otros frameworks que rastrean cambios comparando árboles DOM virtuales completos, myVanilla usa Proxy de ES6 para interceptar modificaciones directamente en el estado del objeto.

DobleParseaArrebatoCremoso.js vincula el parceo del docuento xHtml con el objeto Proxy creado por GargantaProfunda.js

El objeto nativo del componente al integrarlo al navegador guarda un falso virtual-dom solo de los elementos que haga solicitud de datos.

Dicho falso virtual-Dom se referencia en la detecion del Proxy a la hora de detectar que el dato fua modificado.

Si el dato se modifica un elemento se ve impactado directamente sin necesidad de recargar el componente, ni el resto del documento ya que solo esta contenido dentro de esa etiqueta html.

100% Rerctivo Eficiente NO Compilado

Peligro

Hay una extructura de dato que puede ser cambiadas, y se toma en cuenta para el prototipado del componente.

El nombre de la etiqueta que llama al componente app-mi-componente es la ruta de la carpeta del componente componets/mi-componente/app.xml , componets/mi-componente/index.js , componets/mi-componente/style.css

Directivas del Template

Las siguientes directivas te permiten estructurar la lógica directamente en tus atributos xHTML:

Directiva Descripciónes Ejemplos Consideraciones Notaciones
{{"cualquier texto"}} Solo renderiza texto plano entre comillas dobles <span>{{" cualquier texto "}}</span> No deje espacios en blanco entre las llaves { { } } ni como {{ "" }} cualquier texto
if="{{condicion}}" Renderizado condicional basado en la expresión true/false. "condicion" es una propiedad de un objeto o puede ser un metodo. que retorne true o false. <div if="{{isVisible}}";...</div> puede evaluar > , < , >= , <= ,  !== , != , || , && Si la condicion es verdadera se renderiza el elemento y se incluye en el DOM. Pero si es falsa se crea un nodo anchor if y no se renderiza. Pero si tiene el atributo data-anchor = "false" entonce no se crea el nodo anchor ni se renderiza el elemento. Y pierde toda la reactividad.
show="{{condicion}}" Renderizado condicional basado en display none / display flex. "condicion" es una propiedad de un objeto o puede ser un metodo. <div show="{{isVisible}}">...</div> <div show="{{isVisible}}">...</div>

El elemento siempre esta en el nodo. no se crea no se destuye. solo modifica el css display : none ;

module="{{obj.propiedad}}" Atributo que vincula un evento de entrada tipo input y que conecta con el proxy del objeto <input module="{{usuario.nombre}}"/> <p>{{usuario.nombre}}</p> <input module="{{usuario.nombre}}"/>

Es la forma en que se declara la reactividad de los elementos.

Tutorial: trabajando con myVanilla

Explicacion simple y sufuciente para entender como trabaja con myVanilla

Paso 1: Configurando componentes

En myVanilla todo esta basado en componentes. Entonces lo primero que debemos hacer es crear un directorio en la carpeta components, el nombre del directorio sera el nombre de tu componente.

El directorio debe contener un archivo app.xml , index.js style.css

El sistema lo interpreto como un todo del componente. y la reactividad y el sistema de reactividad queda en dicho componente.

A modo de ejemplo es como que cada componente es reactivo por su cuenta y tiene su propio mini-virtual-doom-fake.

Archivo app.xml

Es un arachivo donde vamos a definir el template del componente. y tambien la reactividad

app.xml 
<!-- app.xml -->
<div show="{{usuarios.length > 0}}" onscroll="{{onViewportScroll()}}" style="border:1px solid #ccc; height:300px; overflow-y:scroll; position:relative;">
    <div style="{{heightStyle}}">
        <ul id="lista-benchmark" loop="{{usuario in usuariosVisibles}}" style="{{listaStyle}}">
            <li>
                <span class="id">#{{usuario.id}}</span>
                <strong>{{usuario.nombre}}</strong>
                <span class="badge {{usuario.estado}}">{{usuario.estado}}</span>
                <code>{{usuario.hash}}</code>
            </li>
        </ul>
