# AI Context & Instructions

## Project Overview
This project is built strictly on **Vanilla JavaScript**, HTML5, and CSS3. 

## Architectural Rules
1. **No Frameworks:** Do not use React, Vue, Angular, Svelte, Tailwind, Bootstrap, or any external heavy libraries unless explicitly told.
2. **Directory Structure:**
   - HTML files should reside at the root level or in structured subfolders if the project grows.
   - All CSS files must be inside `assets/css/`.
   - All JS logic must be inside `assets/js/`. The main entry file is `main.js`.
3. **Styling:**
   - Use CSS variables for colors, spacing, and typography to maintain a design system.
   - Implement premium, modern designs (e.g. smooth micro-interactions, clean flexbox/grid layouts). 
4. **JavaScript:**
   - Write clean, modular ES6+ JavaScript.
   - **Strictly use ES6 Classes (Object-Oriented Programming) for all JavaScript logic.** Do not write procedural or purely functional scripts.
   - Always verify DOM elements exist before adding event listeners to avoid console errors.

## General Guidance for AIs
- Prioritize clean diffs and efficient code snippets.
- Use strictly absolute paths for file manipulation if necessary. 
- Keep this file updated if new sweeping conventions are adopted.
- **Google Technologies:** Solo usar tecnologías de Google cuando se pida específicamente.
- **Prohibición de Ejemplos:** Está estrictamente prohibido colocar cualquier tipo de valor como ejemplo (bloques de código de muestra, valores ilustrativos, etc.) dentro de los archivos `.md`. La documentación debe ser 100% teórica.
